## Compatibility Issues

### ⚠️ Decimal Trimming Required

Token has 18 decimals but NTT supports max 8. Amounts will be trimmed, potentially causing precision loss.

**Recommendation**: Solana token will use 8 decimals. Ensure your application handles the decimal difference correctly.

### ⚠️ Self-destruct Capability

Contract contains selfdestruct opcode. If triggered, bridged tokens could become worthless.

**Recommendation**: Review contract for selfdestruct conditions. Ensure it cannot be called maliciously.

### ℹ️ Upgradeable Proxy

Contract is an upgradeable proxy (Some(TransparentUpgradeable)). Implementation can change over time.

**Recommendation**: Monitor for upgrades. NTT integration should be re-verified after any implementation changes.

## Migration Steps

### Step 1: Install NTT CLI

Install the Wormhole NTT CLI tool

```bash
npm install -g @wormhole-foundation/ntt-cli
```

### Step 2: Initialize NTT Project

Create a new NTT deployment configuration

```bash
ntt init
```

### Step 3: Configure Source Chain

Add Ethereum as source chain with Locking mode

```bash
ntt add-chain ethereum --mode locking --token 0x7fc66500c84a76ad7e9c93437bfc5ac33e2ddae9
```

### Step 4: Configure Destination Chain

Add Solana as destination with 8 decimals

```bash
ntt add-chain solana --mode burning --decimals 8
```

### Step 5: Deploy NTT Contracts

Deploy the NTT manager and transceiver contracts

```bash
ntt deploy
```

### Step 6: Configure Rate Limits

Set up rate limits for bridge transfers

```bash
ntt configure-limits
```

### Step 7: Test Transfer

Perform a test transfer with a small amount

```bash
ntt transfer --amount 1 --to <SOLANA_ADDRESS>
```
